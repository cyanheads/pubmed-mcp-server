# 2.6.4 — 2026-04-26

Follow-up to v2.6.3's classification fix. Tool descriptions now declare their cross-field input contracts inline so callers don't learn them from a rejection, and one more imperative throw was lifted into a structured empty result.

## Fixed

- **`pubmed_format_citations` no longer throws when EFetch returns no articles** ([#46](https://github.com/cyanheads/pubmed-mcp-server/issues/46) class). Returns `{ citations: [], totalFormatted: 0, totalSubmitted: N, unavailablePmids }` instead of raising `-32603`. `format()` appends a one-line pointer to `pubmed_search_articles` / `pubmed_spell_check` when the result is empty.

## Changed

- **Top-level descriptions on `pubmed_fetch_fulltext` and `pubmed_lookup_citation` now state the cross-field contract.** The runtime `.refine()` from v2.6.3 catches violations either way; this surfaces the rule upfront. `fetch_fulltext` declares the `pmcids` / `pmids` XOR; `lookup_citation` requires at least one bibliographic field per citation.
- **Field-level `.describe()` audit across 5 tools.** Removes implementation leaks (`pubmed_lookup_citation.candidatePmids` no longer mentions "raw `AMBIGUOUS` CSV"), drops meta-coaching ("downstream code should key on…"), and documents under-specified semantics: `pubmed_convert_ids` outputs (`pmid` / `pmcid` / `doi` / `errmsg`) call out the absent-on-failure / presence-as-signal contract; `pubmed_search_articles.dateRange` documents the form-client empty-string-disables pattern; `pubmed_fetch_fulltext.unavailablePmcIds` corrected to cover both direct-request and PMID-resolved failures.

## Tests

- **462 passed** / 4 skipped. `format-citations.tool.test.ts`: `throws when no articles found` → `returns structured empty result when no articles match`.
